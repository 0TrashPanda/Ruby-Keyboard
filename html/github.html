Website auto deploy script

Immediately after we used FileZilla to deploy our website for the first time, we thought to ourselves: there has to be a better way. After a bit of searching, we found an FTP-Deploy-Action script on GitHub that allows us to automatically deploy our website when we push to our main branch.

You just add this yml script to your actions tab, create some secrets containing the credentials, change the server directory and voilÃ ! A completely autonomous website deploy system.

name: Deploy website on push
on:
  push:
    branches: [ "master" ]
  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

jobs:
  web-deploy:
    name: Deploy
    runs-on: ubuntu-latest
    steps:
    - name: Get latest code
      uses: actions/checkout@v2

        server: ${{ secrets.ftp_host }}
        username: ${{ secrets.ftp_user }}
        password: ${{ secrets.ftp_pass }}
        server-dir: /site/wwwroot/eindwerken/guus_jonah/
