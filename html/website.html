Website

The first thing we worked on was our website, this is partly because it was the only thing we could do but also because

Bootstrap

When we started building our website, we used a bootstrap template. This helped a lot with building out a vision of how we wanted our site to look, of course we still had a lot of work with CSS, icons, images and content. Once we decided that that was indeed the look we were going for, we already worked on it for over 50 hours. Our biggest frustration was how slow the website felt to use, it had a lot of garbage code that we didn’t use and a ton of errors. That’s when we decided to make a lite version of our site. We continued to update the main site with content but stopped doing any visual changes.

Lite site

Instead of doing visual changes on the main site, we worked on making a lite site. We didn’t start from scratch though, we based ourselves on the content of our main site and CSS’ed our way to perfection. We also didn’t want to rewrite our nav-bar and footer on each page so we started using ShadowDOM we also use it to import our documentation from a separate folder and onto our documentation section.

ShadowDOM

Shadow DOM refers to the ability of the browser to include a subtree of DOM elements into the rendering of a document, but not into the main document DOM tree. This means that you can make components and reuse them whenever you would want.

Website hosting

Our website is hosted on an azure web app container that we got access to in the form of a user account and accompanying credentials. When we first got access to the server via for ftp (file transmission protocol) we were instructed to use FileZilla to copy over the files about once a week to update our website… But of course we weren’t happy with that.
